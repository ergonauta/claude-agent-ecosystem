# Claude Agent Ecosystem

A structured multi-agent workflow system for delivering software features with precision,
traceability, and human oversight at every critical decision point.

## Philosophy

> "The plan is the product. The code is the proof."

This ecosystem exists to eliminate the most common failure mode of AI-assisted development:
Claude implementing something plausible but misaligned, then building tests that validate
the wrong behaviour. Every workflow here forces alignment *before* implementation begins.

The human owns the intent. Claude owns the execution. The plan file owns the state.

---

## Core Principles

1. **Specification before implementation** вЂ” Ambiguity is resolved in the plan, not in code
2. **TDD as alignment tool** вЂ” RED tests are a contract, not a formality
3. **Small atomic blocks** вЂ” Each phase is small enough to commit, review, and verify independently
4. **Human in the loop** вЂ” Explicit approval gates at every critical transition
5. **State survives context** вЂ” The plan file is the source of truth across all context clears
6. **ADR-driven decisions** вЂ” Every non-obvious choice is recorded with its rationale

## Installation

Install skills into another project so they become Claude Code slash commands.

### Option A вЂ” Symlinks (recommended)

Skills stay in the ecosystem repo; the target project links to them.
Updates to the ecosystem are picked up immediately.

```bash
git clone <this-repo> ~/tools/claude-agent-ecosystem

# install the default skill set into your project
~/tools/claude-agent-ecosystem/scripts/install.sh /path/to/your-project
```

### Option B вЂ” Git submodule

Pins the ecosystem to a specific commit. Useful for team repos.

```bash
cd /path/to/your-project
git submodule add <this-repo> .claude/ecosystem
.claude/ecosystem/scripts/install.sh . [skill ...]
```

### Selecting skills

```bash
# list available skills
./scripts/install.sh --list

# install specific skills only
./scripts/install.sh /path/to/your-project atomic-commits self-review

# overwrite existing installs
./scripts/install.sh --force /path/to/your-project

# copy files instead of symlinking (when ecosystem won't be on every machine)
./scripts/install.sh --copy /path/to/your-project
```

Skills land in `<target>/.claude/commands/` and are immediately available as `/skill-name` in Claude Code.

## Quick Start

### First time in a new project

```bash
# 1. Run the cartographer to map the codebase
/repo-cartographer

# 2. Start a new feature
/implementation-plan
```

### Starting a feature

1. Run `/implementation-plan` вЂ” Claude enters planning mode and challenges your requirements
2. Review and approve `implementation-plan.md` вЂ” this is your contract
3. Claude creates `implementation-status.md` вЂ” this tracks live state
4. Work through phases with approval gates (see `docs/workflow-detailed.md`)

---

## The Two Files

| File | Purpose | Who writes | Who mutates |
|---|---|---|---|
| `implementation-plan.md` | Intent, spec, acceptance criteria, phase definitions | Human (with Claude challenging) | Frozen after approval |
| `implementation-status.md` | Current phase, completed work, decisions made, ADR notes | Claude (after each phase) | Updated after each human approval |
